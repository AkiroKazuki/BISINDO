# Contributing to BISINDO Emergency Detection

Terima kasih telah tertarik untuk berkontribusi pada proyek ini! Sebagai proyek Karya Tulis Ilmiah (KTI), kami sangat menghargai feedback, laporan bug, dan usulan fitur baru.

## Bagaimana Cara Berkontribusi?

### 1. Melaporkan Bug
Jika Anda menemukan kesalahan teknis atau perilaku yang tidak terduga:
- Buka **Issues** di GitHub.
- Jelaskan bug secara detail, sertakan langkah-langkah untuk mereproduksinya.
- Lampirkan screenshot jika diperlukan.

### 2. Memberikan Saran Fitur
Kami selalu terbuka untuk ide baru guna meningkatkan akurasi atau kegunaan sistem:
- Gunakan fitur **Ideas** atau buka **Issue** baru.
- Jelaskan mengapa fitur tersebut penting dan bagaimana cara kerjanya.

### 3. Kontribusi Kode (Pull Requests)
1. Fork repository ini.
2. Buat branch baru (`git checkout -b fitur-baru`).
3. Lakukan commit pada perubahan Anda (`git commit -m "Tambah fitur X"`).
4. Push ke branch tersebut (`git push origin fitur-baru`).
5. Buat **Pull Request**.

## Standar Kode

- Pastikan kode mengikuti struktur folder yang ada (`bisindo-emergency/ml`, `backend`, `frontend`).
- Berikan komentar pada fungsi yang kompleks.
- Pastikan semua file baru memiliki lisensi yang sesuai (MIT).

## Kode Etik
Harap ikuti [Code of Conduct](CODE_OF_CONDUCT.md) kami dalam setiap interaksi.
